# DESIGN HANDOFF — 파워냅 UI 원칙

실제 프로덕트 레벨의 UI를 목표로 한다. 'AI 티(AI Slop: 무분별한 그라데이션,
조잡한 아이콘 데코레이션, 모호한 정렬)'를 완전히 제거한다.

## AI 티를 제거하는 3대 핵심 원칙 (필수 준수)

1. 필요 없는 모든 장식 제거 (Simplify First)
   - 모노톤 중심의 미니멀 레이아웃에서 시작한다.
   - 텍스트만으로 전달 가능하면 리스트/카드 앞 아이콘을 삭제한다.
     아이콘은 핵심 액션에만 정밀하게 사용한다. 파워냅은 아이콘 0개가 기본.

2. 컨텍스트 및 목적 중심 설계 (Action-Oriented)
   - 사용자가 화면에서 가장 먼저 해야 하는 행동에 3초 안에 시선이 꽂히게 한다.
   - 파워냅의 사용자는 졸린 상태다: 화면당 결정 1개, 핵심 버튼 120pt+.

3. 엄격한 색상 제한 (Color Constraint)
   - 브랜드 컬러 1개(#4353E0) + 무채색 스케일 + 포인트 1개(#E8981F, 커피 전용).
   - 이외의 색 추가 금지. 위계는 폰트 굵기(800/700/600)와 크기로만 잡는다.

## 디자인 시스템 앵커 (확정 — 임의 변경 금지)

- watchOS Glanceable 원칙 = 헌법: 2초 안에 읽고 탭 1번으로 끝난다.
  "이 요소가 애플워치 화면에 들어가는가?"로 모든 추가 요소를 거른다.
- 토스 One Thing per One Page = 정보구조 규칙: 화면당 사용자 결정 1개.
- Apple HIG = 구현 디테일 기준: Dynamic Type 존중, 모든 탭에 햅틱,
  다크모드 자동 대응, 터치 타겟 최소 44pt.
- Material 3에서는 알람 해제 슬라이드 제스처 하나만 차용한다 (§6.3).
- Calm/Headspace류 웰니스 일러스트 스타일 금지. 이 앱은 3초 유틸리티다.

## 캐릭터 허용 구역 (예외 조항)

- Calm/Headspace류 웰니스 일러스트 금지 원칙의 유일한 예외: 수면 화면(sleep.tsx)에만
  자는 강아지 캐릭터를 허용한다. 홈/알람/설정/히스토리 등 다른 화면에는 확장 금지.
- 캐릭터는 장식이 아니라 기존 호흡 애니메이션(숨쉬는 느낌의 scale 반복)을 대체
  하는 기능 요소로만 쓴다 — 별도의 장식용 캐릭터를 추가하는 것이 아님.
