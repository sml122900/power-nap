## 이슈

- 설정 화면을 처음 리팩터할 때 미션 순서 반전을 "슬라이드 = 알람 끄기"로 잘못
  가정하고 구현했다 — 사용자에게 확인 질문을 던져 실제로는 명언 화면까지 알람이
  계속 울려야 한다는 걸 알게 되어 재설계함(상세는 결정 기록 참고).
- jest-expo 환경에서 `await import(...)` 동적 import가 실제로 호출되면
  `--experimental-vm-modules` 없이 호출됨 에러를 낸다는 인프라 갭을 이번 세션에
  두 번(설정 렌더 테스트, 명언 저장 함수 테스트) 재현·확인했다 — 상세는 트러블슈팅
  문서 참고.
- `expo-router`의 파일 기반 라우팅이 `app/` 아래 `.test.tsx` 파일도 그대로
  프로덕션 번들에 포함시켜, 컴포넌트 테스트가 끌어온 Node 전용 모듈(`path`) 때문에
  `expo export`가 깨졌다 — 상세는 트러블슈팅 문서 참고.
- Android 릴리즈 빌드 중 자산 복사 단계(`createBundleReleaseJsAndAssets`)가
  일시적 파일 잠금으로 한 번 실패했다 — 재시도로 바로 해결(코드 이슈 아님, 아마
  백그라운드 안티바이러스 스캔 추정).
