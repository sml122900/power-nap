## 이슈

- 어제 "RevenueCat 키가 빌드에 박혔다"고 보고한 검증이 실제로는 틀린
  방법론(OR 패턴 grep)이었다 — 오늘 사용자가 명시적으로 "값 존재 여부가
  아니라 빌드 주입을 의심하고 실증하라"고 지적해 재검증, 진짜 원인을 찾음.
  교훈: 부정 결과 없음(에러 안 남)을 곧바로 "됐다"로 해석하지 말고, 대조군
  (이미 작동 확인된 다른 값)과 나란히 비교해야 착각을 피할 수 있다.
- `expo export`가 항상 "app/ 아래 테스트 파일" 지뢰를 잡아주는 게 아니다 —
  Node 전용 의존성이 있으면 빌드 시점에 잡히지만, `jest.mock()`처럼 전역
  함수 호출만 있으면 문법적으로 멀쩡해 번들링은 통과하고 런타임에만 터진다.
  기존 트러블슈팅 문서(`expo-router-scans-app-dir-for-tests.md`)가 이미
  "history.test.ts는 운이 좋았을 뿐 구조적으로 똑같이 위험하다"고 예측해뒀던
  게 실제로 몇 주 뒤 재현됨.
- 디버깅 중 반복한 `rm -rf dist`가 실행 중이던 Metro 서버(파일 워처가 `dist/`를
  보고 있었음)를 두 번 죽였다 — 재시작으로 바로 복구했지만, 라이브 Metro
  세션 중엔 export 검증용 `dist/` 삭제를 피해야 한다는 걸 배움.
- adb 기기 인증(`unauthorized`)과 USB 연결 자체가 세션 내내 여러 번 끊겼다 —
  전부 케이블 재연결/재인증으로 해결, 코드 이슈 아님.
- PowerShell의 `>` 리다이렉트가 기본 UTF-16으로 저장돼 Bash `grep`이 내용을
  전혀 못 읽는(모든 문자 사이에 공백처럼 보이는 널바이트) 문제를 겪음 —
  `Out-File -Encoding utf8`로 해결.
- Metro가 기본 8081 대신 8080에 떠 있어서 디버그 빌드가 "Unable to load
  script"로 멈췄다 — `REACT_NATIVE_PACKAGER_HOSTNAME=127.0.0.1` + `adb reverse
  tcp:8081 tcp:8081`로 USB 기반 연결로 전환해 해결(WiFi IP는 그 사이 기기가
  네트워크를 잃어 100% 패킷 손실 상태였음, ping으로 직접 확인).
