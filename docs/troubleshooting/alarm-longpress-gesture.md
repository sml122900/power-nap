# 알람 화면 3초 롱프레스 해제가 실기기에서 전혀 작동하지 않던 문제

## 문제상황

알람 화면의 대체 해제 수단(슬라이드가 어려울 때 손잡이를 3초간 길게 눌러 끄기)이 실기기에서
**전혀** 반응하지 않았다. 코드는 이전 세션에 이미 `Pressable.onLongPress` → `Gesture.LongPress()`로
한 차례 교체된 상태였다(RN 기본 Pressable이 GestureHandlerRootView와 타이밍 레이스를 일으켜
릴리즈 빌드에서만 씹히는 문제 때문에). 그런데도 이번 도그푸딩에서 다시 "완전 미작동"으로
보고됐다.

## 시도한 것들

1. **1차 가설 — 제스처 인식 자체가 너무 엄격함**: `Gesture.LongPress()`의 기본 `maxDistance`
   (약 10pt)는 원래 짧은 롱프레스용 값이라, 3초라는 긴 시간 동안 손가락을 완벽히 고정하기엔
   너무 빡빡하다는 가설을 세웠다. `maxDistance(40)`으로 완화하고, 텍스트 한 줄뿐이던 터치
   영역도 44pt 이상으로 넓혔다(HIG 최소 터치 타겟). 재빌드·재설치 후 재검증 요청.
   → 결과: 여전히 "전혀 작동안하는데?"
2. 사용자에게 "정확히 어느 텍스트를 눌렀는지"를 되물어 확인한 결과, 텍스트 자체를 누르면
   실제로는 **동작한다**는 것을 확인했다 — 즉 1차 수정은 유효했다. 진짜 문제는 기능이
   아니라 **발견성(discoverability)**이었다: 슬라이드 손잡이(사용자가 실제로 만지는 지점)와
   "3초간 눌러 끄기" 안내 텍스트(별도의, 작고 눈에 덜 띄는 위치)가 물리적으로 분리돼 있어서
   사용자가 어디를 눌러야 하는지 몰랐다.

## 최종 해결법

안내 텍스트를 탭 대상에서 완전히 분리하고, 롱프레스 제스처를 **사용자가 이미 밀려고
만지는 슬라이드 손잡이 자체**에 얹었다:

```ts
const thumbGesture = Gesture.Race(pan, longPress);
```

`Gesture.Race`로 두 제스처를 같은 손잡이에 동시에 등록하면, 손가락을 가만히 3초 유지하면
롱프레스가 이기고, 밀기 시작하면 Pan이 활성화되며 롱프레스는 자동으로 취소된다 — 사용자
입장에서는 "이 손잡이 하나로 밀거나 누르거나 둘 다 가능"해진다. 안내 텍스트는 이제 순수
정보 전달용으로만 남겨(밑줄 제거, 제스처 핸들러 없음) 혼동을 없앴다.

실기기 재검증에서 "정상작동해"로 확인됨.

## 이력서 소재 한 줄

기능적으로는 이미 고쳐진 버그를 "안 고쳐졌다"는 사용자 리포트로 오인하지 않고, 재현 조건을
구체적으로 되물어 진짜 원인이 로직이 아니라 인터랙션 설계(탭 대상과 실제 터치 지점의 분리)임을
가려내고, 두 제스처를 하나의 컨트롤에 우선순위 기반으로 결합해 재설계함.
