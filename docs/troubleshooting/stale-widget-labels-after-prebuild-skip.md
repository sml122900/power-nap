# 실기기 위젯에 이름 바꾼 지 오래된 모드 라벨이 남아있던 문제

## 문제상황

낮잠 모드 라벨을 "바로 잠들 것 같아요"/"좀 뒤척일 것 같아요"에서 "매우 졸림"/
"조금 졸림"으로 통일한 지 오래됐는데(커밋 `76c1c02`), 실기기 홈 화면 위젯에는
여전히 옛 라벨이 보인다는 보고가 들어왔다. 코드가 실제로 안 바뀐 건지, 빌드에
반영이 안 된 건지 원인이 불명확했다.

## 시도한 것들

사용자가 지정한 순서대로 하나씩 확인해 어디서 끊겼는지 좁혀갔다:

1. `plugins/withHomeScreenWidgets.js` 소스 확인 — `widget_fast_title`/
   `widget_slow_title`이 이미 "매우 졸림"/"조금 졸림"으로 돼 있고 커밋도 이미
   반영됨. **여기는 문제 없음.**
2. `android/app/src/main/res/values{,-ko}/widget_strings.xml`(prebuild 생성물)
   확인 — 이것도 새 문자열이 이미 들어있었음. **여기도 문제 없음.**
3. 방금 만든 릴리즈 APK를 `aapt2 dump strings`로 직접 열어 문자열 테이블 확인 —
   "매우 졸림"/"조금 졸림"만 있고 옛 문구 흔적이 전혀 없었음. **여기도 정상.**

세 단계 다 정상인데 왜 실기기에서는 옛 문구가 보였을까 — `expo run:android`와
`expo prebuild`의 관계를 다시 짚어봤다.

## 최종 해결법

`expo run:android`는 `android/` 디렉터리가 이미 존재하면 **config plugin을
다시 적용하지 않는다** — `expo prebuild`를 명시적으로 호출해야만 plugin이
재실행된다. 즉 라벨을 바꾼 커밋(`76c1c02`) 이후에도 그냥 `expo run:android`로만
빌드했다면, 그 커밋 *이전*에 생성된 `android/`를 그대로 재사용해 옛 문자열이
계속 나왔을 것이다.

이 세션에서는 마침 별개의 릴리즈 서명 작업 때문에 `expo prebuild --clean`을
이미 돌렸고, 그 결과 android/가 최신 plugin 소스로 재생성되면서 이 문제가
부수적으로 이미 해결돼 있었다 — 코드 수정은 필요 없었고, 원인 규명과
CLAUDE.md 지뢰 기록만 남았다.

## 이력서 소재 한 줄

"코드는 맞는데 실기기 반영이 안 됨" 유형의 버그를 소스 → 빌드 생성물 →
최종 산출물 순으로 단계별 직접 확인해 코드 문제가 아님을 먼저 배제하고,
빌드 도구 체인(config plugin이 `prebuild`에만 재적용되고 `run:android`
단독 실행으로는 재적용 안 됨)의 동작 방식을 근본 원인으로 특정함.
